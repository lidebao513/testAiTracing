## 产品简介

本地LLM代码生成与评估系统是一个基于本地大语言模型的代码生成和评估工具，能够根据用户提示生成高质量的代码，并对生成的代码进行全面评估。系统集成了Ollama API、LangSmith跟踪系统和RAGAS评估框架，实现了从代码生成到结果验证的完整流程。

**主要功能：**
- 基于本地大语言模型的代码生成
- 模型故障自动切换机制
- 详细的代码评估和质量分析
- 完整的测试覆盖系统
- 代码生成和评估过程的详细跟踪

## 安装指南

### 步骤1：安装Python

1. 访问 [Python官网](https://www.python.org/downloads/) 下载并安装Python 3.10或更高版本
2. 安装过程中勾选"Add Python to PATH"
3. 验证安装：打开命令行，输入 `python --version` 确认Python版本

### 步骤2：安装Ollama

1. 访问 [Ollama官网](https://ollama.com/) 下载并安装Ollama
2. 启动Ollama服务
3. 下载所需模型：
   ```bash
   ollama pull deepseek-r1:1.5b
   ollama pull deepseek-r1:7b
   ```

### 步骤3：克隆项目代码

1. 打开命令行，导航到您想要安装项目的目录
2. 执行以下命令克隆项目：
   ```bash
   git clone <项目仓库地址>
   cd testAiTracing
   ```

### 步骤4：安装项目依赖

1. 创建虚拟环境：
   ```bash
   python -m venv venv
   ```

2. 激活虚拟环境：
   - Windows: `venv\Scripts\activate`
   - macOS/Linux: `source venv/bin/activate`

3. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

### 步骤5：配置环境变量

1. 在项目根目录创建 `.env` 文件
2. 添加以下环境变量：
   ```
   LANGCHAIN_API_KEY=your_langsmith_api_key
   LANGCHAIN_PROJECT=your_project_name
   LANGCHAIN_TRACING_V2=true
   LANGCHAIN_ENDPOINT=https://api.smith.langchain.com
   ```

## 基本操作流程

### 生成代码

1. **准备提示词**：准备清晰的代码生成提示词，例如："生成一个计算斐波那契数列的Python函数"

2. **使用代码生成功能**：
   ```python
   from code_generator import generate_code
   
   prompt = "生成一个计算斐波那契数列的Python函数"
   template = "{prompt}"
   code = generate_code(prompt, template)
   print(code)
   ```

3. **保存生成的代码**：将生成的代码保存到文件中

4. **执行代码**：运行生成的代码并验证结果

### 评估代码

1. **使用RAGAS评估**：
   ```bash
   python ragas_with_langsmith.py
   ```

2. **查看评估结果**：评估结果将显示在命令行中，并上传到LangSmith

3. **分析评估报告**：登录LangSmith查看详细的评估报告和跟踪信息

## 高级功能使用

### 模型故障自动切换

系统会自动检测模型状态，当首选模型（deepseek-r1:1.5b）不可用时，会自动切换到备用模型（deepseek-r1:7b）。

### 自定义代码生成模板

您可以创建自定义的代码生成模板，以获得更符合特定需求的代码：

```python
from code_generator import generate_code

prompt = "生成一个线性回归模型"
template = "# {prompt}\n\nimport numpy as np\n"
code = generate_code(prompt, template)
print(code)
```

### 测试覆盖分析

运行测试覆盖分析，确保代码质量：

```bash
# 运行语句覆盖测试
py -m pytest test_statement_coverage.py -v --cov=code_generator --cov-report=html

# 运行分支覆盖测试
py -m pytest test_branch_coverage.py -v --cov=code_generator --cov-report=html

# 运行条件覆盖测试
py -m pytest test_condition_coverage.py -v --cov=retry_decider --cov-report=html
```

### 集成测试

运行端到端集成测试，验证系统各组件的协作：

```bash
py -m pytest test_integration.py -v
```

## 常见问题排查

### 问题1：Ollama服务无法启动

**症状**：运行代码时出现"连接超时"错误

**解决方案**：
1. 确认Ollama服务已启动
2. 检查网络连接
3. 验证Ollama端口（默认11434）是否被占用
4. 尝试重启Ollama服务

### 问题2：模型生成的代码语法错误

**症状**：生成的代码包含语法错误

**解决方案**：
1. 提供更清晰、更具体的提示词
2. 尝试使用不同的模型（deepseek-r1:7b）
3. 检查生成的代码是否符合Python语法规范

### 问题3：LangSmith跟踪失败

**症状**：无法在LangSmith中查看跟踪信息

**解决方案**：
1. 确认.env文件中的环境变量配置正确
2. 验证LangSmith API密钥有效
3. 检查网络连接是否正常

### 问题4：测试覆盖率低

**症状**：测试覆盖率报告显示覆盖率低于预期

**解决方案**：
1. 检查测试用例是否覆盖所有代码路径
2. 添加更多测试用例，特别是边界情况
3. 运行 `py -m pytest --cov-report=html` 查看详细的覆盖率报告

## 常见问题解答

### Q: 系统支持哪些编程语言？

A: 系统主要支持Python代码生成，但也可以尝试生成其他编程语言的代码，如JavaScript、Java、C++等。

### Q: 如何提高代码生成的质量？

A: 提高代码生成质量的方法包括：
- 提供详细、具体的提示词
- 使用合适的模板
- 选择适合任务的模型
- 对生成的代码进行人工审查

### Q: 系统需要互联网连接吗？

A: 基本的代码生成功能不需要互联网连接，因为模型运行在本地。但LangSmith跟踪和某些评估功能需要互联网连接。

### Q: 如何自定义评估指标？

A: 您可以修改 `ragas_with_langsmith.py` 文件，添加或修改评估指标，以满足特定的评估需求。

### Q: 系统的响应速度如何？

A: 响应速度取决于硬件配置和模型大小。在推荐配置下，代码生成通常需要1-5秒，评估需要5-10秒。

### Q: 如何扩展系统支持更多模型？

A: 您可以修改 `local_llm_config.py` 文件，添加新的模型配置和故障转移逻辑。

## 术语表

### LLM
大语言模型（Large Language Model），是一种能够理解和生成人类语言的人工智能模型。

### Ollama
一个用于在本地运行大语言模型的工具，支持多种模型的部署和管理。

### LangSmith
一个用于跟踪、调试和评估LLM应用的平台，提供详细的执行跟踪和评估功能。

### RAGAS
一个用于评估检索增强生成（RAG）系统的框架，提供多种评估指标。

### DeepEval
一个用于评估LLM生成结果的框架，提供质量评估功能。

### 测试覆盖率
衡量测试用例覆盖代码的程度，包括语句覆盖、分支覆盖和条件覆盖等。

### 故障转移
当首选系统或服务不可用时，自动切换到备用系统或服务的机制。

### 提示词
用户提供给LLM的指令或问题，指导模型生成特定的输出。

### 模板
预设的文本结构，用于指导代码生成的格式和内容。
