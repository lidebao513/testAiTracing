# testAiTracing 系统操作方式手册

## 1. 项目概述

### 1.1 项目背景

testAiTracing 是一个基于本地大语言模型的 RAG（检索增强生成）系统评估平台，旨在提供全面的 RAG 应用性能评估和质量监控。该系统集成了 Ollama 本地模型、LangSmith 追踪系统和 RAGAS 评估框架，实现了从 RAG 应用测试到性能评估的完整流程。

### 1.2 主要功能

- **RAGApp API 健康检查**：验证 RAG 应用服务的可用性和响应状态
- **RAGClient 模块**：与 RAG 应用 API 进行交互，执行查询操作
- **RAGAS 评估**：对 RAG 应用的生成结果进行多维度评估
- **DeepEval 门禁检查**：执行质量门禁检查，确保生成结果的可靠性
- **LangSmith 追踪**：对评估过程进行详细的追踪和记录
- **集成验证**：对整个系统进行全面的集成测试和验证

### 1.4 关键特性

- **模型故障自动切换**：当首选模型不可用时，自动切换到备用模型
- **详细的评估指标**：包括忠实度、回答相关性、上下文精确率和召回率
- **完整的追踪系统**：通过 LangSmith 实现评估过程的详细追踪
- **自动化测试流程**：提供完整的集成测试和验证脚本
- **多维度评估**：从多个角度评估 RAG 应用的性能

## 3. 安装部署步骤

### 3.1 安装Python

1. 访问 [Python官网](https://www.python.org/downloads/) 下载并安装Python 3.10或更高版本
2. 安装过程中勾选"Add Python to PATH"
3. 验证安装：打开命令行，输入 `python --version` 确认Python版本

### 3.2 安装Ollama

1. 访问 [Ollama官网](https://ollama.com/) 下载并安装Ollama
2. 启动Ollama服务
3. 下载所需模型：
   ```bash
   ollama pull deepseek-r1:1.5b
   ollama pull deepseek-r1:7b
   ```

### 3.3 克隆项目代码

1. 打开命令行，导航到您想要安装项目的目录
2. 执行以下命令克隆项目：
   ```bash
   git clone <项目仓库地址>
   cd testAiTracing
   ```

### 3.4 安装项目依赖

1. 创建虚拟环境：
   ```bash
   python -m venv venv
   ```

2. 激活虚拟环境：
   - Windows: `venv\Scripts\activate`
   - macOS/Linux: `source venv/bin/activate`

3. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

### 3.5 配置环境变量

1. 在项目根目录创建 `.env` 文件
2. 添加以下环境变量：
   ```
   # 启用 LangSmith 跟踪
   LANGCHAIN_TRACING_V2=true
   
   # LangSmith API 密钥
   LANGCHAIN_API_KEY=your_langsmith_api_key
   
   # LangSmith 项目名称
   LANGCHAIN_PROJECT=test_rag_evaluation
   
   # RAG 应用 API URL
   RAG_APP_API_URL=http://localhost:8000
   ```

## 4. 核心功能操作指南

### 4.1 RAGApp API 健康检查

**功能说明**：验证 RAG 应用服务的可用性和响应状态

**使用方法**：
```python
from scripts.rag_client import RAGClient

# 初始化客户端
client = RAGClient()

# 执行健康检查
is_healthy = client.health_check()
print(f"服务状态: {'正常' if is_healthy else '异常'}")
```

### 4.2 RAGClient 查询操作

**功能说明**：与 RAG 应用 API 进行交互，执行查询操作

**使用方法**：
```python
from scripts.rag_client import RAGClient

# 初始化客户端
client = RAGClient()

# 执行查询
question = "什么是 RAG 技术？"
result = client.ask(question, k=2, temperature=0.3)

# 查看结果
print(f"问题: {question}")
print(f"回答: {result['answer']}")
print(f"来源: {len(result['sources'])} 个文档")
```

### 4.3 RAGAS 评估

**功能说明**：对 RAG 应用的生成结果进行多维度评估

**使用方法**：
```bash
# 运行 RAGAS 评估脚本
python tests/integration/test_ragas_with_tracing.py
```

**评估指标**：
- 忠实度（faithfulness）：评估回答是否忠实于上下文
- 回答相关性（answer_relevancy）：评估回答与问题的相关性
- 上下文精确率（context_precision）：评估上下文与问题的相关性
- 上下文召回率（context_recall）：评估上下文是否包含回答问题所需的所有信息

### 4.4 DeepEval 门禁检查

**功能说明**：执行质量门禁检查，确保生成结果的可靠性

**使用方法**：
```bash
# 运行 DeepEval 门禁脚本
python tests/integration/test_deepeval_gate_on_ragapp.py
```

### 4.5 集成验证

**功能说明**：对整个系统进行全面的集成测试和验证

**使用方法**：
```bash
# 运行集成验证脚本
python verify_integration.py --verbose --output integration_report
```

**验证内容**：
- RAGApp API 可访问性验证
- RAGClient 模块功能验证
- RAGAS 评估脚本执行验证
- DeepEval 门禁脚本执行验证
- LangSmith 环境变量配置验证

## 6. 常见问题解决方法

### 6.1 服务连接问题

**问题**：无法连接到 RAGApp 服务

**解决方案**：
1. 确认 RAGApp 服务已启动
2. 检查 `RAG_APP_API_URL` 配置是否正确
3. 验证网络连接是否正常
4. 尝试重启 RAGApp 服务

### 6.2 模型相关问题

**问题**：Ollama 模型连接失败

**解决方案**：
1. 确认 Ollama 服务已启动
2. 检查模型是否已正确下载
3. 尝试使用不同的模型（如从 deepseek-r1:1.5b 切换到 deepseek-r1:7b）
4. 验证 Ollama 端口（默认 11434）是否被占用

### 6.3 评估相关问题

**问题**：RAGAS 评估执行失败

**解决方案**：
1. 确认 Ollama 服务和 RAGApp 服务都已启动
2. 检查评估脚本中的模型配置
3. 验证网络连接是否正常
4. 尝试增加评估超时时间

### 6.4 配置相关问题

**问题**：LangSmith 追踪失败

**解决方案**：
1. 确认 `.env` 文件中的环境变量配置正确
2. 验证 LangSmith API 密钥有效
3. 检查网络连接是否正常
4. 确认 `LANGCHAIN_TRACING_V2` 已设置为 `true`

## 7. 注意事项

### 7.1 性能优化

- **模型选择**：根据硬件配置选择合适的模型，小模型速度快但质量可能较低
- **批量评估**：对于大量评估任务，建议分批执行以避免内存不足
- **缓存机制**：利用 RAGAS 的缓存机制减少重复评估
- **并行处理**：对于多核心系统，可以考虑并行执行评估任务

### 7.2 安全注意事项

- **API 密钥保护**：不要将 LangSmith API 密钥硬编码在代码中
- **网络安全**：确保 RAGApp 服务只在安全的网络环境中运行
- **数据隐私**：评估过程中使用的测试数据应避免包含敏感信息
- **权限控制**：限制对评估系统的访问权限

### 7.3 最佳实践

- **定期评估**：定期执行集成验证，确保系统性能稳定
- **模型更新**：及时更新 Ollama 模型以获得更好的性能
- **监控日志**：定期查看评估日志，及时发现和解决问题
- **文档维护**：保持系统文档的更新，确保团队成员了解系统使用方法
- **版本控制**：使用 Git 进行版本控制，跟踪系统变更

## 8. 术语表

### 8.1 核心术语

| 术语 | 解释 |
|------|------|
| RAG | 检索增强生成（Retrieval-Augmented Generation），结合信息检索和生成式 AI 的方法 |
| LLM | 大语言模型（Large Language Model），能够理解和生成人类语言的人工智能模型 |
| Ollama | 用于在本地运行大语言模型的工具，支持多种模型的部署和管理 |
| LangSmith | 用于跟踪、调试和评估 LLM 应用的平台，提供详细的执行跟踪和评估功能 |
| RAGAS | 用于评估检索增强生成（RAG）系统的框架，提供多种评估指标 |
| DeepEval | 用于评估 LLM 生成结果的框架，提供质量评估功能 |
| 门禁检查 | 确保系统输出质量的验证机制，通常设置质量阈值 |
| 追踪 | 记录系统执行过程的详细信息，用于调试和分析 |
| 评估指标 | 用于衡量系统性能的量化标准，如忠实度、相关性等 |

### 8.2 技术术语

| 术语 | 解释 |
|------|------|
| 忠实度（faithfulness） | 评估回答是否忠实于上下文信息的指标 |
| 回答相关性（answer_relevancy） | 评估回答与问题相关性的指标 |
| 上下文精确率（context_precision） | 评估检索到的上下文与问题相关性的指标 |
| 上下文召回率（context_recall） | 评估检索到的上下文是否包含回答问题所需全部信息的指标 |
| 故障转移（failover） | 当首选系统不可用时，自动切换到备用系统的机制 |
| 虚拟环境（virtual environment） | 隔离的 Python 运行环境，用于管理项目依赖 |
| 环境变量（environment variable） | 存储配置信息的系统变量，如 API 密钥等 |
| 集成测试（integration test） | 验证系统各组件协作是否正常的测试方法 |
