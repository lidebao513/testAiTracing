## 3. 特殊文件的处理方式

### 3.1 配置文件

- **环境配置文件**：`.env` 文件包含敏感信息，应放在 `config/env/` 目录，并确保不被版本控制
- **pytest配置**：`pytest.ini` 可以放在项目根目录或 `config/pytest/` 目录
- **GitHub Actions配置**：保持在 `.github/workflows/` 目录不变

### 3.2 第三方依赖

- **虚拟环境**：`venv/` 和 `test_env/` 目录不应纳入版本控制，应在 `.gitignore` 中排除
- **依赖列表**：`requirements.txt` 应保持在项目根目录，便于安装依赖

### 3.3 文档文件

- **README.md**：可以保持在根目录，也可以放在 `docs/` 目录并在根目录创建链接
- **中文文档**：统一放在 `docs/` 目录下，按类型分类

### 3.4 日志和临时文件

- **日志文件**：统一放在 `logs/` 目录，并在 `.gitignore` 中排除
- **临时文件**：统一放在 `temp/` 目录，并在 `.gitignore` 中排除
- **覆盖率报告**：统一放在 `coverage_report/` 目录

### 3.5 缓存和编译文件

- **__pycache__/**：应在 `.gitignore` 中排除
- **.coverage**：应在 `.gitignore` 中排除
- **.deepeval/**：DeepEval配置目录，可以保留

### 4.2 注意事项

#### 4.2.1 导入路径更新

**重要**：移动文件后，需要更新所有Python文件中的导入路径。

**示例：**
```python
# 原导入语句
from code_generator import generate_code
from local_llm_config import LocalLLMConfig

# 新导入语句
from src.core.code_generator import generate_code
from src.llm.local_llm_config import LocalLLMConfig
```

**建议**：在每个子目录下创建 `__init__.py` 文件，简化导入路径。

#### 4.2.2 配置文件路径更新

**pytest.ini**：
```ini
[pytest]
testpaths = tests
python_files = test_*.py
python_classes = Test*
python_functions = test_*
```

**GitHub Actions**：
```yaml
- name: Run tests
  run: python -m pytest tests/ -v
```

#### 4.2.3 虚拟环境处理

- 不要移动虚拟环境目录（`venv/`, `test_env/`）
- 如果需要重新创建虚拟环境，删除后重新创建即可

#### 4.2.4 Git版本控制

- 使用 `git mv` 命令移动文件，保留Git历史
- 更新 `.gitignore` 文件，排除不需要版本控制的文件

#### 4.2.5 测试验证

整理完成后，运行以下命令验证：
```bash

python -m pytest tests/ -v

# 检查导入是否正常
python -c "from src.core.code_generator import generate_code; print('导入成功')"

```
