# 本地LLM代码生成与评估系统 - 项目文档包

## 1. 项目概述

**项目名称**：本地LLM代码生成与评估系统
**项目类型**：人工智能工具开发
**项目时长**：2周
**项目目标**：构建一个基于本地大语言模型的代码生成和评估系统，实现从代码生成到结果验证的完整流程。

**项目价值**：
- 减少对云端LLM服务的依赖，提高数据安全性
- 提供高质量的代码生成能力，提升开发效率
- 实现全面的代码评估和测试覆盖，确保代码质量
- 为开发团队提供可定制的AI辅助开发工具

## 5. 技术挑战与解决方案

### 5.1 挑战1：本地LLM模型调用稳定性

**挑战**：本地模型调用可能面临连接超时、模型未找到等问题，影响系统稳定性。

**解决方案**：
- 实现了模型故障自动切换机制，当首选模型不可用时，自动切换到备用模型
- 设计了多层次错误检测机制，能够准确识别各类调用失败场景
- 配置了合理的重试策略，包含重试次数和重试间隔参数

**技术实现**：
```python
# 模型故障自动切换逻辑
def get_llm(model_name: str = "deepseek-r1:1.5b") -> BaseLLM:
    """获取LLM实例，支持故障转移"""
    try:
        # 尝试使用首选模型
        return Ollama(model=model_name)
    except Exception as e:
        logger.warning(f"模型 {model_name} 调用失败: {str(e)}")
        # 故障转移到备用模型
        logger.info("尝试使用备用模型 deepseek-r1:7b")
        return Ollama(model="deepseek-r1:7b")
```

### 5.2 挑战2：代码生成质量控制

**挑战**：生成的代码可能存在语法错误、逻辑错误或不符合最佳实践的问题。

**解决方案**：
- 实现了代码语法验证功能，确保生成的代码符合语法规范
- 集成了RAGAS评估框架，对生成的代码进行全面评估
- 设计了代码执行和结果验证机制，确保代码功能正确

**技术实现**：
```python
# 代码语法验证
def validate_syntax(code: str) -> bool:
    """验证代码语法"""
    try:
        compile(code, '<string>', 'exec')
        return True
    except SyntaxError:
        return False
```

### 5.3 挑战3：测试覆盖完整性

**挑战**：确保所有代码路径都被测试覆盖，特别是边界情况和异常处理。

**解决方案**：
- 实现了多种测试覆盖策略，包括语句覆盖、分支覆盖和条件覆盖
- 设计了全面的测试用例，覆盖正常情况、边界情况和异常场景
- 使用pytest-cov工具，实时监控测试覆盖率，确保达到90%以上

**技术实现**：
```bash
# 测试覆盖率分析命令
py -m pytest test_statement_coverage.py -v --cov=code_generator --cov-report=html
```
